数学公式 Mathjax

前情提要

写大雾实验不确定度分析教程这篇博客的时候用到了数学公式
而next自带的渲染器不支持latex语法
尽管vscode书写时可以预览，但是网页上不能显示出来
#### 解决过程

  1. 上网搜索，最终选择了pandoc作为数学公式的渲染器
  2. 首先安装pandoc
  3. 更换渲染器
  4. 修改配置 如果将per_page设置为true，需要在每一篇需要渲染数学公式的博客最前面添加一行
    使公式能够渲染
    而设为false，则自动渲染每一篇博客，不需要再前面添加
    个人想保证整体的加载速度就选择了per_page: true
  5. hexo clean && hexo g -d生成并部署即可

Tips

  • 如果在渲染的时候出现报错
    说明没有安装/没有合适版本的pandoc，推荐用homebrew下载
  • pandoc渲染基本完全支持latex语法，但在书写的时候存在一些需要注意的细节，比如标题换行后得到正文，需要在标题行最后打两个空格

Large files

前情提要

我怎么就不小心hexo g -d了超过100m的大文件
部署了很久，然而显示

就算真的部署完毕了也不能成功，显示

解决过程

  1. 首先当然是把本地文件夹中的大文件删了。但是没有用，还是会进行部署
    于是复制了报错信息，上网寻找教程，使用
    命令，发现输出
    说明提交的目标分支内文件没有变
    于是使用git log命令查看提交内容，发现没有输出
    很头秃。这表示我并没有提交任何东西到github上
  2. 继续上网找教程，发现都是git commit相关，并没有提及hexo d的背景
    修改关键词之后查找，找到一两篇hexo提交大文件的相关内容，其中提到了用于部署的.deploy_git文件夹
    而我在我的博客根目录下没有找到该目录
    又卡了一下。但是在终端键入cd .deploy_git命令，发现是可以进去的 然后使用git log，跳出多次提交记录，按q退出 好…明白了，意思是这个目录才是真正提交到github中的
    于是找到大文件对应的提交记录，执行重置命令 跳出信息 应该找对路了
    然后进行hexo clean && hexo g -d
    还是不行？？报和之前一样的错误
  3. 最后直接把.deploy_git文件夹一整个删除
    cd blog进入博客根目录，执行hexo clean && hexo g -d
    等了很久…因为是重建整个部署页面
    最终居然成功了
    然后结束
